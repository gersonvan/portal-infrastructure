# Instruções para Prompts e Desenvolvimento

Este documento serve como guia central para todas as interações e desenvolvimento do projeto. Ele define as regras, padrões e práticas que devem ser seguidas durante o desenvolvimento do novo Portal Administrativo.

## Regras Gerais de Documentação

1. **Instruções de Prompt**
    - Dê preferência a me mostrar o que vai fazer antes de executar, isso evitará retrabalhos caso eu não tenha explicado direito
    - Pode me questionar. Apesar de dar sugestões, nem sempre eu trago a melhor maneira de se pensar, gosto de ser questionado

2. **Organização de Arquivos**
   - Cada módulo deve ter seu próprio diretório em `/docs/modules/`
   - Documentos específicos do módulo devem ficar em seu respectivo diretório
   - Documentação geral do projeto fica na raiz de `/docs/`

3. **Nomenclatura**
   - Usar kebab-case para nomes de diretórios (ex: `diploma-digital`)
   - Usar SNAKE_CASE para nomes de arquivos principais (ex: `PROMPT_INSTRUCTIONS.md`)
   - Usar PascalCase para nomes de classes e componentes
   - Usar camelCase para nomes de funções e variáveis

4. **Estrutura de Documentos**
   - Todo documento deve começar com um cabeçalho descritivo
   - Usar títulos hierárquicos adequadamente (# para título principal, ## para seções, etc)
   - Incluir seção de status/progresso quando aplicável
